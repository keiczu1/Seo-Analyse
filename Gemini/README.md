# Gemini pages

## Semantic HTML verification

The semantic HTML refactor was completed on 2026-06-26. Source files live under `src/`, `styles/`, `scripts/`, `assets/`, and `tools/`; root HTML files are regenerated browser entry points.

`npm.cmd run gemini:verify` checks the semantic HTML contract:

- exactly one `main` and one `h1` per generated page;
- `header.site-header`, `main`, and `footer.site-footer` are present;
- repeated navigation landmarks have accessible names;
- breadcrumbs keep the named `nav` landmark generated by the build tool;
- visible `input`, `textarea`, and `select` controls have accessible names;
- standalone `section` elements have a heading or accessible name;
- content-section headings do not skip levels, while independent `aside`, `footer`, `header`, and `nav` components are ignored for that heading-level rule;
- existing local-reference, text-corruption, mobile-menu, focus/hover, and horizontal-overflow checks stay active.

Use this verification chain before publishing Gemini changes:

```powershell
npm.cmd run gemini:build
npm.cmd run gemini:audit-css
npm.cmd run gemini:verify
git diff --check -- Gemini package.json
```

Эта папка содержит собранные статические страницы LawDivorce и исходники для их генерации.

## Что открывать в браузере

Открывать нужно HTML-файлы из корня `Gemini`:

- `index.html`
- `zadat-vopros.html`
- `category-divorce.html`
- `article-divorce-without-consent.html`
- `article-ui-kit.html`
- `all-components-ui-kit.html`

Не открывайте файлы из `src/pages/` напрямую. Это шаблоны для сборщика. В них есть директивы `@include` и `@render`, которые браузер не понимает.

## Структура

- `AGENTS.md` - правила для будущих AI-агентов: где источник истины, что открывать, что не превращать обратно в один большой HTML.
- `src/pages/` - шаблоны страниц.
- `src/partials/` - общие части: head, header, footer, модальные окна, article-блоки.
- `src/data/` - данные навигации, футера, категорий, юристов и страниц.
- `styles/` - модульные CSS-файлы.
- `scripts/` - модульные JS-файлы.
- `assets/` - изображения, иконки и документы.
- `tools/` - сборка, аудит CSS, smoke-проверка и служебные утилиты.

## Сборка

Запускать из корня репозитория:

```powershell
npm.cmd run gemini:build
```

Команда пересобирает корневые HTML-файлы из `src/pages/`.

## Внутренние UI kit страницы

- `article-ui-kit.html` - рабочая страница для будущего UI kit компонентов статьи.
- `all-components-ui-kit.html` - полный справочник всех доступных компонентов LawDivorce. Он создан как клон текущего расширенного `article-ui-kit.html`, чтобы сохранить общую витрину перед дальнейшим сужением article kit.

## Проверка

Запускать из корня репозитория:

```powershell
npm.cmd run gemini:audit-css
npm.cmd run gemini:verify
```

`gemini:verify` может создать временные файлы в `reports/`. Это генерируемый вывод, он игнорируется и не должен храниться как часть чистой папки.

## Правило чистой папки

Оставляем только исходники, собранные HTML, assets, styles, scripts, tools и документацию.

Не храним скриншоты, visual-review dumps, smoke-отчеты и разовые comparison-файлы. Временные артефакты можно складывать в `reports/`, но после проверки их нужно удалять.
