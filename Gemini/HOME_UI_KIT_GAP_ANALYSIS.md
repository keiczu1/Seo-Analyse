# Home components missing from article UI kit

Дата анализа: 2026-06-26.

## Цель

Нужно было сравнить `Gemini/index.html` с `Gemini/article-ui-kit.html` и найти элементы главной страницы, которые могут пригодиться повторно, но не были представлены в UI kit. Persistent-правки выполняются в source-слое: `Gemini/src/pages/article-ui-kit.html`, `Gemini/styles/`, `Gemini/src/data/pages.json`; root HTML обновляется только через `npm.cmd run gemini:build`.

## Что уже было в UI kit

`article-ui-kit.html` до доработки хорошо покрывал статью:

- дизайн-токены;
- текстовую разметку;
- лид статьи, summary card и choice cards;
- callout-блоки;
- таблицы, статусы, сценарии и судебную практику;
- steps/checklist для статьи;
- документы и скачивание;
- FAQ, правовую базу, автора, дисклеймер и перелинковку.

## Что было только на главной

В `index.html` обнаружена отдельная homepage-система с префиксом `hp-*` и чат-компонентами `tg-*`. Эти элементы не были представлены в UI kit:

- hero live badges: онлайн-юристы, среднее время ответа, актуальность материала;
- hero boundary block: понятное разделение бесплатной консультации и платных услуг;
- Telegram-like chat widget: сценарные вкладки, шапка юриста, пузырьки сообщений, CTA-инпут;
- channel cards: письменное обращение и телефонная консультация с разделителем `или`;
- trust strip: компактные карточки доверия;
- homepage steps timeline и CTA после шагов;
- responsibility cards: бесплатно / обсуждается отдельно;
- interstitial CTA strip;
- category search и category cards;
- lawyer cards с аватаром, статусом, рейтингом и микроотзывом;
- recent consult cards;
- verification timeline и verification stats badge;
- review cards;
- preparation checklist с progress bar и safety warning;
- final CTA section.

## Решение

UI kit не должен становиться второй копией главной. Добавлен один логичный раздел `Компоненты главной страницы`, где показаны компактные образцы каждого reusable-паттерна:

- `Лид, живые бейджи и чат-виджет`;
- `Карточки списков и социальное доказательство`;
- `Доверие, процесс и проверка специалистов`;
- `Подготовка к консультации и финальный CTA`.

Для отображения production-компонентов к внутренней странице UI kit подключён `styles/pages/home.css` через `Gemini/src/data/pages.json`. Локальные правила в `styles/pages/ui-kit.css` только ограничивают витрину внутри article layout и делают `.hp-reveal` видимым без запуска домашнего scroll-аниматора.

## Разделение UI kit страниц 2026-06-26

Текущий расширенный UI kit сохранён отдельной внутренней страницей `all-components-ui-kit.html`. Это полный каталог компонентов LawDivorce: статья, главная, карточки, CTA, доверие, формы, FAQ и навигационные блоки.

`article-ui-kit.html` оставлен как рабочая страница для следующего этапа: из неё можно будет убрать всё, что не относится к оформлению самой статьи, не теряя общий справочник компонентов.

## Browser review 2026-06-26

После первого добавления раздел был проверен в браузере через Playwright. Скриншоты сохранены во временную папку `Gemini/reports/ui-kit-review/`.

Найденные проблемы:

- блок `Компоненты главной страницы` читался как плотная галерея, а не как справочник;
- разные по смыслу компоненты стояли в одной сетке: карточка направления, юрист, консультация и отзыв конкурировали друг с другом;
- на мобильной ширине вводные карточки сжимались в две слишком узкие колонки;
- разделу не хватало явной логики: зачем эти блоки идут именно в таком порядке.

Исправленная логика подачи:

1. Сначала объясняется принцип: переносим reusable-паттерны, но не копируем весь лендинг.
2. Затем идёт карта раздела: вход и выбор канала, доверие, списки/доказательства, подготовка и финальный CTA.
3. Демонстрации идут крупными последовательными блоками, а не плотной сеткой.
4. На мобильной ширине вводные и демонстрационные блоки переходят в одну колонку.
5. `home.css` сужен: правило `overflow-x` больше не цепляет глобальный `html`, чтобы подключение home styles к UI kit не влияло на всю страницу.

## Что не переносилось

- Весь лендинг главной целиком: это сделало бы UI kit нечитаемым.
- Полная интерактивность `home.js`: для справочника достаточно статических состояний; production-поведение остаётся на главной.
- Все 12 направлений и все 4 юриста: в UI kit достаточно одного representative-примера карточки каждого типа.
