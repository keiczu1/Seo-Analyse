# Stage 3. Fact-Check + Score

Цель этапа - не пропустить в HTML непроверенные факты и не перейти к Stage 4, пока бриф не набрал проходной score.

## Gate

Переход к Stage 4 запрещен, пока Stage 3 не завершен. После проверки выведи блок:

```text
=== STAGE 3 COMPLETE ===
Score: XX/44
Fact-check: пройдена | частичная (N фактов unverified)
Переход к Stage 4: разрешен / запрещен (причина)
========================
```

Перед генерацией HTML запроси подтверждение пользователя, если score ниже production-ready или остались `unverified` факты.

## Fact-check

Через WebSearch проверь:

1. Числовые данные: госпошлины, сроки, штрафы, лимиты, проценты.
2. Номера статей и нормативных актов.
3. Актуальность: были ли изменения в текущем году.
4. URL источников: ссылка ведет на реальную страницу, а не на заглушку.
5. Расхождения между источниками: выбери более свежий официальный источник и укажи альтернативу.

Приоритет источников: официальный сайт ведомства > КонсультантПлюс/Гарант > профильные СМИ. Если факт не подтвержден, пометь его как `unverified` и не превращай в уверенное утверждение.

## Score

Оцени бриф по 22 критериям, 0-2 балла каждый, максимум 44:

- 38-44 - production-ready;
- 32-37 - можно использовать после правок;
- меньше 32 - вернуться к Stage 2.

Критичные группы проверки: intent fit, page type, first screen, UX-модули, фактическая точность, качество источников, покрытие SEO-триплетов, schema, редакторские правила, YMYL safeguards, юридические компоненты.

## Benchmark

Если в проекте есть `benchmark/golden-queries.json`, сравни score с эталоном. Если score ниже эталона, укажи дельту и конкретные секции для усиления.
